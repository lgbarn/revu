The happy path is well built and all available automated gates pass, but revu does not yet satisfy its own hostile-input and CLI contracts. Executive answer

revu does the basic Git diff, render, search, theme, fold, and clean-quit flows well. It does not consistently do what its README and security policy promise under adversarial filenames, malformed repo config, ordinary unified patches, nested working directories, multi-commit pager input, no-newline diffs, wrapped lines, or very large diffs. Shipping should wait for the two security blockers and the high-severity behavior gaps below.

Confirmed blockers and high-severity findings

SeverityFindingEvidence and trigger
BLOCK Repository filename can become an editor command. The active diff path is appended directly to $VISUAL/$EDITOR without an option terminator. With the default Vim-compatible editor, a root filename such as +!touch PWN is an Ex command, not a file. Pressing e executes it. src/app.rs:986-997. This directly contradicts SECURITY.md:36-39, which says user paths are separated with --.
BLOCK Untrusted repo-local theme can panic. parse_hex accepts any six-byte body, then byte-slices at 0..2, 2..4, and 4..6. A valid TOML value such as two three-byte Unicode characters passes the length check and panics at a non-character boundary. src/theme.rs:502-516; repo config is loaded from src/config.rs:135-138,231-237. This violates the untrusted-config threat model in SECURITY.md:40-42.
HIGH Saved state defeats explicit CLI flags. CLI overrides are applied during config resolution, then state.json overwrites line numbers, wrapping, and hunk headers. --no-line-numbers can therefore show line numbers, despite the help and README saying the CLI wins. src/config.rs:160-179 then src/app.rs:246-281; contract at README.md:127-139,159-162.
HIGH Nested-directory operation uses incompatible path bases. Git diff paths are repository-root-relative, but editor and blame subprocesses run from the caller's current directory. Blame silently disappears and e opens the wrong file. Untracked discovery from a subdirectory is scoped differently and omits untracked files elsewhere in the repo. src/vcs/git.rs:40-47,72-110,165-210; src/app.rs:986-997,1291-1305. A direct Git fixture confirmed git diff emitted sub/file while blame from sub/ interpreted it as sub/sub/file.
HIGH Ordinary unified patches render as no changes. The parser creates a file only after diff --git. A standard diff -u patch beginning with ---/+++/@@ creates a hunk but no file, so the hunk is discarded. src/diff.rs:85-157,219-222; advertised patch support at README.md:35-37,94-95. Claude independently flagged the orphan-hunk drop.
HIGH git log -p corrupts multi-commit input. After the first hunk, later commit headers are classified as context lines until the next diff --git, corrupting content and line numbers. The README explicitly advertises this pager flow. src/diff.rs:92-145; contract at README.md:98-103.
HIGH Wrapped content cannot be navigated reliably. Ratatui scrolls wrapped visual rows, while revu calculates bounds, active file, selection, and navigation from unwrapped logical line counts. One long wrapped line can extend below the viewport while max_offset remains zero. src/app.rs:547-593,1138-1145. This breaks the advertised --wrap mode.

No-newline markers are treated as real context. \ No newline at end of file advances both counters, breaks remove/add pairing, and appears on both sides in split/vertical modes.src/diff.rs:124-141; counter behavior at src/render.rs:247-279,487-496,622-683.

Wide glyphs misalign columns and tabs have zero terminal width, effectively disappearing from code.src/render.rs:843-851,904-933; src/app.rs:1505-1545. Both models independently flagged this.

Cross-model result

Both independent passes found

  • SIGTERM/SIGHUP and panic mouse-capture restoration gaps.
  • Incorrect character-count display width for wide glyphs and tabs.
  • No lower bound for live_interval_ms.
  • The focused network-crate denylist is not a proof that future code remains offline.
  • Documentation drift around live refresh and the stale 0.1.x security policy.
  • Non-Git orphan hunks are discarded and state writes are non-atomic.

Codex-only findings from the independent stage

The editor option injection, Unicode theme panic, CLI/state precedence inversion, nested-directory path-base mismatch, multi-commit pager corruption, wrapped-scroll bounds, no-newline marker corruption, 65,535-line ceiling, pure-rename path regression, and missing stash metadata were not found by Claude. These are source-confirmed and drive the final BLOCK verdict.

Claude-only findings from the independent stage

  • gh pr diff is strict UTF-8 while local Git paths use lossy decoding: src/app.rs:121-133.
  • Pager mode has no stdout-TTY guard, so redirection can write terminal control output to a file: src/pager.rs:23-38.
  • The latent blame ref argument lacks --end-of-options: src/vcs/git.rs:165-176. Current app callers pass no ref, so it is not presently reachable.
  • Word-diff runtime is unbounded for pathological lines: src/worddiff.rs:86-111.

Model disagreement

Claude's independent verdict was conditional SHIP and stated that no high-severity issues or hostile-input panics were reachable. The fixed byte slicing in parse_hex, editor argv construction, and the documented parser/path contracts disprove that conclusion. The final cross-referenced verdict is therefore BLOCK.

Attack vectors to retain as regression tests

  1. Clone a repo with a root file named like a Vim + command, review it, and press e. Verify only the file opens and no command runs.
  2. Place a six-byte non-ASCII hex body in .revu/config.toml. Verify a clean validation error, never a panic.
  3. Set saved state opposite to each explicit display flag. Verify the explicit CLI value wins.
  4. Run from a nested subdirectory with changed and untracked files inside and outside it. Verify diff inclusion, blame, reload, and editor paths.
  5. Feed standard diff -u, multi-commit git log -p, no-newline markers, a pure rename with spaces, an absolute two-file diff, and a stash. Assert the parsed model and rendered metadata.
  6. Render a single line that wraps to more rows than the viewport and a model above 65,535 rows. Verify every visual row remains reachable.
  7. Render tabs, combining marks, CJK, and emoji in split mode. Assert separator columns by terminal display width.
  8. Send SIGTERM and induce a controlled panic after mouse capture. Verify cooked mode, normal screen, visible cursor, and mouse capture disabled.
